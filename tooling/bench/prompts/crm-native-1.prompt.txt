Build a lightweight CRM in NocoBase, FULL AUTONOMY — no questions, do it end to end. Build into env **expagents** — every nb api call uses `-e expagents -y`. The instance is a CLEAN empty NocoBase.

DATA MODEL — snake_case collections; belongsTo/relation fields named as NOUNS (e.g. `customer`, `contact`, `owner`), NEVER ending in `_id` (beta rejects `*_id` association names):
- crm_customers: name, industry (single-select enum), size (enum: SMB / Mid-Market / Enterprise), website, status (enum: prospect / active / churned), owner (belongsTo users)
- crm_contacts: name, email, phone, title, customer (belongsTo crm_customers), owner (belongsTo users)
- crm_deals: title, customer (belongsTo crm_customers), contact (belongsTo crm_contacts), stage (enum: lead / qualified / proposal / negotiation / won / lost), amount (number, ¥ prefix, 2 decimals), expected_close (date), owner (belongsTo users)
- crm_activities: subject, type (enum: call / email / meeting / task), due (date), done (checkbox), deal (belongsTo crm_deals), contact (belongsTo crm_contacts)

SEED all-English data: ≥8 customers, ≥12 contacts, ≥16 deals covering EVERY stage (incl. won & lost), ≥24 activities covering every type, with realistic owners/amounts/dates.

PAGES — use native NocoBase blocks only (Table / Kanban / Calendar / Details / Form / Chart). Put everything under a top-level "CRM" menu group. Plan per page:

1. Dashboard — a sales overview built from NATIVE Chart blocks (use the Chart builder block):
   - Deals count by stage (bar or funnel)
   - Pipeline amount by stage, or amount by expected_close month (bar/line)
   - Deals by owner (pie)
   - plus a Table block "Closing soon" (crm_deals sorted by expected_close ascending, stage not won/lost) and/or "Recent activities".
2. Pipeline — a Kanban of crm_deals grouped by `stage` (cards show title + customer + amount), AND a Table view of all deals with Filter + Add. Row/card click → drawer: deal Details + an Activities sub-table (that deal) + the linked contact.
3. Customers — Table + Filter + Add new. Row click → a WIDE drawer: customer Details + a Contacts sub-table + a Deals sub-table (both scoped to that customer).
4. Contacts — Table (with a Customer column) + Filter + Add. Row click → drawer: Details + an Activities sub-table.
5. Activities — a Calendar view (by `due`) AND a Table; Filter by type and by done; row click → drawer with Details.

Build it like a real product CRM a NocoBase consultant would ship — clean, populated, navigable. Sub-tables in drawers must auto-filter to the parent record. When done, report the page schemaUid(s) and a one-line summary of what each page contains.
